Vera's First Programming Project in D3

Is there a correlation between the marital status and the number of children that people in Sweden have?

Great thanks go to Victor Armash who helped me a lot on the programming side of this data visualisation project.

Data source: World Value Survey

Analytical trail with WVS visualisation tools

Step 1: Choose the time period.

Step 2: Select the countries to focus on.

Step 3: Select the survey variables 'how many children do you have'.

The image shows the corresponding data table that contains data transformations such as summing, mean, standard deviation and percentages. The bar chart contains visual mappings such as color to number of children and height to the number of respondents in percentages. The data types are categorical and numerical.

Step 4: Choose the crossing variable 'marital status'.

Coosing the crossing variable shows a data table for each possible combination of marital status and children. The data can be transformed into percentages when selected.

In the bar chart, the color is mapped to the marital status, the height of the bar to the number of respondents and the number of children to the horizontal positioning of the bar.

Improvement of the WVS visualization tools

Adding view transformations and modifying the visual structures

The data display in parallel coordinates allows to filter the data according to different variables eg. '2 children' and 'divorced' and gives an approximate number of respondents. The user can move filters around, enlarge them and thereby focus on different categorical data. The data view transformations provided by WVS were limited, no zooming or filtering was possible and the barcharts were not big enough to identify the differences. All data is shown in one graph which makes it hard to read and draw conclusions from.

Limitations of the visualization and future work

The visualisation with parallel coordinates allows to filter for different conditions and see how they connect to each other. However, this tool is not helpful for summing up the total amount of people with one child for example as the scale for number of people is detailed enough. Ideally, both a marital status and a number of children should be selected in order to compare and analyse different lines.

This visualization could be improved by adding hovereffects so that when the mouse is moved over a line, its data is shown in a little box. Also it could have been nice to map one line to one respondent of the survey so that the graph would contain over 1000 different lines. Furthermore, the sample size (around 1000 people) might not be big enough so that insightful conclusions can be drawn on the correlation of marital status and children. Further statistical calculations would be necessary to validate it. More variables should be added as axes of the parallel coordinates to analyse their influence on the two variables like for example income or education level. Showing only two variables introduces a worldview gap by not showing enough informaton as to draw significant conclusions from it. However, it was not possible for me to include this amount of detail into the graph. It would have been necessary to filter the raw data for different conditions eg married Swedes with one children of lowest income and education level and so forth until all possible combinations of the variables are fullfilled. This could be an interesting research approach for future work.
